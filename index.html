ctrlp.vim

Full path fuzzy file, buffer, mru and tag finder for Vim.

  • Written in pure Vimscript for MacVim and Vim 7.0+.
  • Full support for Vim’s regexp as search pattern.
  • Built-in Most Recently Used (MRU) files monitoring and search.
  • Built-in root directory finder.
  • Open multiple files at once.
  • Create new files and directories.
  • Execute Ex commands on an opening file (jump to a line, a string or do anything).
  • Optional cross-sessions caching and history allow for fast initialization.
  • Mappings and usage conform to Vim’s conventions.

Screenshot: CtrlP in regex and full path mode with 3 files selected to open.

Basic Usage:

  • Press <c-p> or run :CtrlP or :CtrlP [starting-directory] to invoke CtrlP.
  • Or run :CtrlPBuffer or :CtrlPMRU to start in find buffer or find MRU mode, respectively.
  • Or run :CtrlPMixed to search in Files, Buffers and MRU files at the same time.
  • Check :help ctrlp-extensions for commands provided by extensions.
Once CtrlP is open:
  • Press <c-f> and <c-b> to cycle between modes.
  • Press <c-d> to switch to filename search instead of full path.
  • Press <c-r> to switch to regexp mode.
  • Press <F5> to purge the cache for the current directory and get new files.
  • Submit / or \ as the input string to find and go to the project’s root.
  • Submit two or more dots . to go backward the directory tree by one or multiple levels.
  • End the input with a colon : followed by a command to execute it after opening the file:
    • Use :45 to jump to line 45.
    • Use :/any\:string to jump to the first instance of any:string.
    • Use :+setf\ myfiletype|25 to set the filetype to myfiletype then jump to line 25.
    • Use :difft when opening files marked by <c-z> to run :difft on the first 4 files.
  • Use <c-n>, <c-p> to go to the next/previous string in the prompt’s history.
  • Use <c-y> to create a new file and its parent directories.
  • Use <c-z> to mark/unmark multiple files and <c-o> to open them.

Basic Options:

  • Change the mapping to invoke CtrlP and the default command it uses: 
    let g:ctrlp_map = '<c-p>'  " Leave this empty to disable the default mapping
let g:ctrlp_cmd = 'CtrlP'
  • When invoked, unless a starting directory is specified, CtrlP will automatically set its local working directory according to this variable: 
    let g:ctrlp_working_path_mode = 2
    0 - don’t manage working directory.
    1 - the directory of the current file.
    2 - the nearest ancestor that contains one of these directories or files:
    .git/ .hg/ .svn/ .bzr/ _darcs/ or your own marker_dir/ marker_file
  • To exclude files or directories from the listing, use the Vim’s option wildignore: 
    set wildignore+=*/tmp/*,*.so,*.swp,*.zip   " Linux/MacOSX
set wildignore+=tmp\*,*.swp,*.zip,*.exe    " Windows
    And/Or the built-in option g:ctrlp_custom_ignore: 
    let g:ctrlp_custom_ignore = '\.git$\|\.hg$\|\.svn$'
let g:ctrlp_custom_ignore = {
	\ 'dir':  '\.git$\|\.hg$\|\.svn$',
	\ 'file': '\.exe$\|\.so$\|\.dll$',
	\ 'link': 'SOME_BAD_SYMBOLIC_LINKS',
	\ }
  • Specify an external tool to use for listing files instead of using Vim’s globpath(). Use %s in place of the target directory: 
    let g:ctrlp_user_command = 'find %s -type f'        " MacOSX/Linux
let g:ctrlp_user_command = 'dir %s /-n /b /s /a-d'  " Windows
    Single VCS listing command: 
    let g:ctrlp_user_command = ['.git/', 'cd %s && git ls-files']
let g:ctrlp_user_command = ['.git/', 'cd %s && git ls-files', 'find %s -type f']
let g:ctrlp_user_command = ['.hg/', 'hg --cwd %s locate -I .']
    Multiple VCS listing commands: 
    let g:ctrlp_user_command = {
	\ 'types': {
		\ 1: ['.git/', 'cd %s && git ls-files'],
		\ 2: ['.hg/', 'hg --cwd %s locate -I .'],
		\ },
	\ 'fallback': 'find %s -type f'
	\ }

Run :help ctrlp or submit ? in CtrlP for more mappings, commands and options.
